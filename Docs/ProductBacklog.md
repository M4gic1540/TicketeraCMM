# 1. Introducción

## 1.1 Propósito

Este documento contiene el Product Backlog de Agile Service Desk Platform (ASDP), organizado por épicas y funcionalidades.

El backlog representa el conjunto priorizado de funcionalidades que permitirán construir la plataforma de manera incremental siguiendo Scrum.

---

# 2. Priorización

Se utilizará la siguiente clasificación.

| Prioridad | Descripción |
|-----------|-------------|
| 🔴 Crítica | Imprescindible para el funcionamiento del sistema |
| 🟠 Alta | Funcionalidad prioritaria para el MVP |
| 🟡 Media | Agrega valor, pero puede esperar |
| 🟢 Baja | Evolución futura |

# 4. Definición de Ready (DoR)

Una historia estará lista para desarrollarse cuando:

- Esté claramente descrita.
- Tenga criterios de aceptación.
- Esté estimada.
- No tenga dependencias bloqueantes.
- Haya sido revisada por el Product Owner.

---

# 5. Definición de Done (DoD)

Una historia se considerará completada cuando:

- El código esté implementado.
- Las pruebas unitarias sean exitosas.
- Las pruebas de integración sean exitosas.
- El Pull Request haya sido aprobado.
- La documentación esté actualizada.
- El despliegue en el entorno correspondiente sea exitoso.
- El Product Owner acepte la funcionalidad.
